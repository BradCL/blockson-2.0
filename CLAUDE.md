# Working in this repo (agent guide)

Blockson is a static-site engine: one engine, many client sites, each described by
a single `content.json` plus an image folder. The engine validates that file and
emits SEO-ready static HTML. A deterministic patch system lets a non-technical
owner make only safe, reversible edits. Read `SPEC.md` for the invariants — they
are the contract the code and the proof suite are written against.

## Build & test

- `npm test` — the proof suite (the quality gate; must stay green).
- `npm run test:overlay` — browser smoke for the click-to-edit overlay.
- `npm run test:editor` — browser smoke for the owner editor's keep-open flow.
- `npm run build:all` — build the example clients.
- `npm run serve <client>` — run the owner editor locally against a client.

Browser smokes need the Playwright Chromium and are not part of `npm test`; run
them when you touch the overlay or the editor UI.

## Documentation discipline (read before you push)

Docs are split into **frozen** and **live**. This split exists because the frozen
docs are large teaching snapshots (the learning guide) and capture-driven
tutorials (screenshots/transcripts regenerated by Playwright harnesses) — keeping
them in lock-step with every change costs far more than it's worth, and a reader
consumes them as coherent wholes, not changelogs.

- **Frozen** (carry a "❄️ Frozen doc" banner — do NOT edit per change):
  `docs/learning-guide/`, `docs/tutorial/developer/`, `docs/tutorial/owner/`.
- **Live** (normative/reference — keep accurate, update with the change that
  affects them): `SPEC.md`, `OPERATOR.md`, `BLOCK_CATALOG.md`.

**Before every push, assess documentation impact and act accordingly:**

1. If the change alters behavior or structure that a **live** doc documents →
   update that live doc **in the same change** (so code never ships ahead of its
   reference).
2. If it affects something a **frozen** doc teaches → add a one-line entry under
   that doc in [`docs/DEFERRED_DOC_UPDATES.md`](docs/DEFERRED_DOC_UPDATES.md)
   (what changed + which chapter/capture). Do **not** edit the frozen doc.
3. If neither → push freely; no doc action is needed. Do not manufacture empty
   ledger entries.

`docs/DEFERRED_DOC_UPDATES.md` is the ledger of what the frozen docs owe; it is
applied in periodic bundled reconciliation passes, never piecemeal. Each ledger
section records a baseline commit, so `git log <baseline>..HEAD -- engine/` is the
raw delta if an entry was ever missed.

## Commits

- Imperative, owner-benefit-framed subject lines (see `git log`).
- Do not add a `Co-Authored-By` trailer.
- Commit/push only what the task calls for; leave unrelated untracked paths alone.
